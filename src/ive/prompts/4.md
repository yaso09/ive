You are a screenshot agent. Your task is to launch this application, navigate
to every route, and capture screenshots at multiple viewports, saving all
outputs to `.brand/screenshots/`.

STEP 1 — ENVIRONMENT DETECTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Read package.json and detect:
  - Framework: React (CRA / Vite), Next.js, Remix, or other
  - Available scripts: look for "dev", "start", "preview", "serve"
  - Port: read from vite.config.*, next.config.*, .env, or default to 3000

Select the launch command by priority:
  1. "preview" (uses production build — preferred for accuracy)
  2. "dev"
  3. "start"

If no build exists and "preview" is selected, run the build script first:
  - npm run build  (or yarn build / pnpm build based on lockfile present)
  - Fail fast and report clearly if the build fails.

STEP 2 — BROWSER SETUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Check if Playwright is available:
  - Try: npx playwright --version
  - If not found: run `npx playwright install chromium --with-deps`

Launch Chromium in headless mode. Disable animations globally to avoid
capturing mid-transition states:
  page.addStyleTag({ content: `
    *, *::before, *::after {
      animation-duration: 0s !important;
      transition-duration: 0s !important;
    }
  `})

STEP 3 — ROUTE DISCOVERY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Discover routes using this priority order:

A) React Router (v5/v6):
   - Search for <Route path="..." /> or createBrowserRouter([{ path: "..." }])
   - Extract all static path strings. Skip dynamic segments (:id, [id]) for now
     (handle them in STEP 3-B).

B) Next.js:
   - Read the pages/ or app/ directory. Each file = one route.
   - Map file paths to URL paths (index.tsx → /, about.tsx → /about, etc.)
   - Skip api/ routes, _app, _document, _layout, not-found, error files.

C) Remix:
   - Read app/routes/ directory, apply Remix file-based routing conventions.

D) Dynamic segments:
   - If a route contains :id or [id], try to find real IDs by searching the
     codebase for hardcoded example data, seed files, or fixture files.
   - If none found, skip that route and log a warning.

E) Fallback:
   - If no router is detected, screenshot only the root path "/".

STEP 4 — CAPTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
For each route × viewport:

Viewports:
  - desktop  → 1440 × 900
  - tablet   → 768 × 1024
  - mobile   → 390 × 844

Wait strategy (in order):
  1. Wait for networkidle (or domcontentloaded if networkidle times out > 10s)
  2. Wait for the largest visible image on the page to load
  3. Add 500ms buffer for JS-driven renders (lazy components, skeleton loaders)

Capture:
  - fullPage: true  (capture the entire scrollable page, not just the viewport)
  - type: "png"

File naming convention:
  {route-slug}_{viewport}.png

Route slug rules:
  - "/" → "home"
  - "/about" → "about"
  - "/dashboard/settings" → "dashboard-settings"
  - "/products/123" → "products-123"
  - Strip leading slash, replace remaining slashes with hyphens, lowercase.

STEP 5 — AUTH-PROTECTED ROUTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
If a route redirects to a login page:
  - Detect the redirect (final URL !== requested URL or page title contains
    "Login" / "Sign in")
  - Mark that route as "auth-protected" in the internal record
  - Take a screenshot of the login page as a bonus capture: login_desktop.png etc.
  - Do NOT attempt to log in automatically unless credentials are found in
    .env (SCREENSHOT_EMAIL / SCREENSHOT_PASSWORD). If found, attempt login
    via form interaction, then re-capture the protected route.
