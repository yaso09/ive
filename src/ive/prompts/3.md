GENERAL RENDERING RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Each output SVG must be fully self-contained: no external fonts, no
  external images, no references to CSS files.
- Embed fonts as base64 @font-face declarations inside a <defs><style> block
  if the component uses a non-system font. If you cannot embed the font,
  fall back to the closest system font and add a comment noting the substitution.
- All pixel values must be unitless inside SVG attributes (SVG user units).
- Use a viewBox that tightly fits the rendered content with 16px padding on
  all sides. Set width and height attributes to match the viewBox dimensions.
- Do not use foreignObject. Represent everything with native SVG elements.

──────────────────────────────────────────────────────
div/section with bg-color   → <rect fill="..." rx="..." />
border-radius               → rx / ry on <rect>
border                      → stroke + stroke-width on <rect>
box-shadow                  → <filter> with feDropShadow
text node                   → <text> with font-family, font-size, fill
placeholder/input text      → <text> styled as input appearance
inline <svg> icon           → copy the SVG subtree directly into a <g>
raster image (<img>)        → <image> with base64 href (xlink:href fallback)
horizontal rule             → <line>
flex row (side-by-side)     → manual x-offset calculation between children
flex column (stacked)       → manual y-offset calculation between children
gap / margin / padding      → translate children accordingly

Each HTML file may contain multiple <section data-variant="..."> blocks.
Render every variant into the same SVG file:
- Arrange variants vertically with 32px gap between them.
- Wrap each variant in a <g id="variant-{name}"> group.
- Add a <text> label above each group: variant name in 11px gray (#888).
- The viewBox height must accommodate all variants stacked.

Example structure:
  <svg viewBox="0 0 240 180" ...>
    <defs><style>/* fonts */</style></defs>

    <!-- variant: default -->
    <text x="16" y="16" font-size="11" fill="#888">default</text>
    <g id="variant-default" transform="translate(16, 24)">
      ...
    </g>

    <!-- variant: danger -->
    <text x="16" y="90" font-size="11" fill="#888">danger</text>
    <g id="variant-danger" transform="translate(16, 98)">
      ...
    </g>
  </svg>
