PART A — ASSET USAGE ANALYSIS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
For each file in `.brand/assets/`, find every source file that references it:

Search strategies (apply all):
  1. Exact filename match — grep for the base filename across all .ts, .tsx,
     .js, .jsx, .css, .scss, .html files
  2. Import path match — resolve the import to an absolute path and compare
  3. Public folder reference — if the file lives in /public, also search for
     its URL-relative path (e.g., "/logo.png", "./logo.png")

For each asset, determine its role using these heuristics:
  "logo"         — filename contains "logo", "brand", "wordmark"
  "icon"         — filename contains "icon", "ico", "favicon", or file is
                   ≤ 64×64 px SVG/PNG
  "illustration" — filename contains "illustration", "hero", "banner",
                   "graphic", or file is a large decorative image
  "background"   — used in CSS background-image or className="bg-..."
  "avatar"       — filename contains "avatar", "profile", "user", "person"
  "thumbnail"    — filename contains "thumb", "preview", "card"
  "sound-effect" — audio file used in response to a user action
  "ambient"      — audio file used as background/loop
  "unknown"      — none of the above match
